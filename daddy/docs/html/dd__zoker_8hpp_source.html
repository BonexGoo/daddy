15 // ■ 토큰구조
16 // - key: [utf8스트링:N]
17 // - group-nameable: [타입:1][자식수량:1+ → L][점퍼규격:1 → N] + ([점퍼:N][점퍼:N]...L개)
18 // - group-indexable: [타입:1][자식수량:1+ → L][점퍼규격:1 → N] + ([점퍼:N][점퍼:N]...L개)
19 // - value-string: [타입:1][사이즈:1+ → N][utf8스트링:N]
20 // - value-binary: [타입:1][사이즈:1+ → N][바이너리덤프:N]
21 // - value-int8: [타입:1][정수:1]
22 // - value-int16: [타입:1][정수:2]
23 // - value-int32: [타입:1][정수:4]
24 // - value-int64: [타입:1][정수:8]
25 // - value-uint8: [타입:1][부호없는 정수:1]
26 // - value-uint16: [타입:1][부호없는 정수:2]
27 // - value-uint32: [타입:1][부호없는 정수:4]
28 // - value-uint64: [타입:1][부호없는 정수:8]
29 // - value-float32: [타입:1][실수:4]
30 // - value-float64: [타입:1][실수:8]
31 //
32 // ■ 토큰관계성
33 // - 시작토큰은 value/group만 OK
34 // - key: 하위로 key만 NO (value/group만 OK)
35 // - group-nameable: 하위로 key만 OK
36 // - group-indexable: 하위로 key만 NO (value/group만 OK)
37 // - value: 하위로 모두가 NO (종착지점)
38 //
39 // ■ 부연설명
40 // - value는 원본 그대로 (파싱비용 ZERO, 스트링은 null포함)
41 // - group은 점퍼라는 seek-table방식으로 하위접근 (nameable은 이진탐색)
42 // - "1+"는 가변길이 정수타입 (8비트중 7비트만 사용, 1비트는 연장플래그)
